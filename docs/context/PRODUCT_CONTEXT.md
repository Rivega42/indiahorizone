# Product Context

> Что мы строим, для кого, и почему именно так. Pre-read для любого AI-агента или нового члена команды.

## Что

**IndiaHorizone** — tech-enabled India concierge для русскоязычных клиентов. Мы продаём **уверенность путешествовать в Индии без хаоса**, а не туры.

## Для кого

Целевая аудитория phase 3:

| Сегмент | % | Что хочет | Что боится |
|---|---|---|---|
| **Wellness-путешественник** (30-45 лет) | ~40% | Аюрведа, йога, тишина, разгрузка | Гастрит, обман на ресепшене, языковой барьер |
| **Adventure-турист** (25-40 лет) | ~30% | Гималаи, треки, фотогенично | Травмы, плохая страховка, потеряться в горах |
| **Cultural-explorer** (35-55 лет) | ~20% | Храмы, города, история, фестивали | Туристические ловушки, толпа, разочарование низким качеством |
| **Family / friends group** (35-50 лет) | ~10% | Контролируемый experience для группы | Ответственность за всех, разные интересы |

Все они: **русскоговорящие**, **не первый раз в путешествиях** (но часто первый — в Индии), **готовы платить премиум за peace of mind**.

## Почему именно так

### Почему «без хаоса» — главное

Индия — sensory overload для нового туриста. Наш differentiator: **снимаем 80% мелких трений** (трансферы, языковой барьер, выбор еды, понимание местной валюты, обращение в полицию при необходимости) → клиент тратит энергию на впечатления, не на логистику.

### Почему «русскоязычный»

- Российская аудитория после 2022: ограничены destination-options + visa-rules + payment-channels
- Доверие к русскоязычному support 24/7 значительно выше чем к английскому
- Нет конкурентов с полным русским workflow (продажи + ops + tech + legal в РФ)

### Почему «tech-enabled»

- Конкуренты: рекламные турагенты с manual-flow (telegram + email). Не масштабируются после 50 клиентов.
- Наше: документация + чат + SOS + дневник в одном app → клиент не тратит время на чаты с разными людьми.
- AI-ассистент с context маршрута (phase 4) — отвечает на «можно ли есть рис в этом отеле» в 19:00 в Ришикеше без human latency.

### Почему НЕ туроператор (де-юре)

Туроператор по 132-ФЗ требует финансовое обеспечение (фингарантии 500к+) и реестровую запись. На phase 3 мы — **турагент** (агентский договор) + **concierge** (отдельные доп.услуги). Юридическая граница в `docs/LEGAL/TOUR_OPERATOR.md`.

## Бизнес-модель

См. `docs/BUSINESS_MODEL/UNIT_ECONOMICS.md`. Кратко:

- Avg ticket: 250-320к ₽ за тур (10-14 дней)
- Margin: 30-40% (наземка в Индии 35-50% от ticket)
- LTV/CAC: предполагаемый 3:1 на phase 3 (дискаунт через рефералы → low CAC)
- Path to PMF: 12+ платящих клиентов / квартал = signal что modul работает

## Конкуренты

- **Western luxury concierge** (Black Tomato, Original Travel, Scott Dunn): английский, цена в 3-5× выше нашей
- **Российские большие агентства** (Кругозор, БГ-Тур): мass-market, нет русскоязычного 24/7 support'а
- **Индивидуальные гиды-русскоговорящие в Индии**: нет tech, нет договорной защиты для клиента
- **Tilt** мы между luxury concierge и агентствами — premium personalization за middle price

## Зрелость продукта

| Аспект | Phase 3 | Phase 4-5 |
|---|---|---|
| **Tech** | Web + PWA, SOS-кнопка через web push | Native mobile, offline-first |
| **Ops** | SOS на Шивам'е (узкое место) | 2+ concierge с playbooks (P0 #278) |
| **Sales** | Manual через Telegram | CRM + AI quote drafts |
| **Caталог** | 10-15 туров через JSON seed | Admin UI + 30+ туров |
| **Платежи** | РФ юрлицо → IN PVT LTD через SWIFT | Online checkout + multi-channel |
