# E2E User Stories — внешний сайт + личный кабинет

> **Назначение.** Сценарии для **end-to-end тестирования** клиентских поверхностей IndiaHorizone (внешний сайт `apps/web` без авторизации + личный кабинет после логина).
>
> **Отличие от [`USER_STORIES.md`](../USER_STORIES.md).** Тот файл — продуктовые истории по JTBD (бизнес-acceptance). Этот — **техническое тест-покрытие**: каждая история привязана к конкретному API-эндпоинту и UI-маршруту, описывает измеримый поведенческий контракт «нажал → API позвался → результат отрисовался».
>
> **Как использовать тестировщику.**
> 1. Прогоняй истории по разделам в порядке follow-the-flow (регистрация → профиль → поездка → ...).
> 2. Каждая история = один тест-кейс (пометь ✅/❌/⏭).
> 3. При расхождении BE↔FE — заведи bug-issue со ссылкой на ID истории (`US-EXT-04`, `US-CAB-12` и т.п.).
> 4. Поле **«Текущий статус»** обновляется по мере доведения фронта до бэка (см. [`BE_FE_CONVERGENCE_PLAN.md`](./BE_FE_CONVERGENCE_PLAN.md)).
>
> **Легенда статуса:**
> - 🟢 **READY** — UI отрисован, API подключён, e2e-сценарий проходит.
> - 🟡 **PARTIAL** — либо UI без API (mock/static), либо API без UI, либо известные баги.
> - 🔴 **MISSING** — нет ни UI, ни подключения; нужно реализовать.
>
> **Источник:** запрос Roman, 2026-05-02. Версия: v0.1.
