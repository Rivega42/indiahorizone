# Онбординг клиента — от лида до старта поездки

> Закрывает [#38](https://github.com/Rivega42/indiahorizone/issues/38). Часть EPIC 4 [#31](https://github.com/Rivega42/indiahorizone/issues/31).
> Статус: Draft v0.1.

## Этапы

```
Лид → Квалификация → Квиз → КП → Договор → Предоплата → Активная подготовка → 
За 7 дней → За 1 день → Старт поездки
```

## Чек-лист по этапам

### 1. Лид (входящий) — sales

- [ ] Зафиксирован источник (Telegram / реф / реклама)
- [ ] Первый ответ ≤ SLA (см. [`SLA.md`](./SLA.md))
- [ ] Создана карточка клиента в CRM
- [ ] Назначен ответственный sales

### 2. Квалификация — sales

- [ ] Подтверждены: даты, бюджет, кол-во людей, цели поездки
- [ ] Зафиксированы особые обстоятельства (дети, аллергии, маломобильность)
- [ ] Решение: подходит / не подходит (если не подходит — вежливый отказ + рекомендация)

### 3. Квиз — клиент

- [ ] Отправлена ссылка на квиз
- [ ] Клиент заполнил
- [ ] Результат в CRM, sales уведомлён

### 4. КП — sales + Shivam

- [ ] Маршрут собран в Notion / Google Doc
- [ ] Себестоимость согласована с Shivam
- [ ] Маржа применена
- [ ] Невозвратные позиции явно указаны
- [ ] КП отправлен клиенту (PDF + комментарий)
- [ ] Созвон по КП назначен (или выяснение в чате)

### 5. Договор + предоплата — sales + finance

- [ ] Подписан договор / акцептована оферта
- [ ] **Согласие на ПДн** получено ([`docs/LEGAL/CONSENTS/PDN.md`](../LEGAL/CONSENTS/PDN.md), [#52](https://github.com/Rivega42/indiahorizone/issues/52))
- [ ] **AML-проверка пройдена** ([`docs/FINANCE/PAYMENTS/AML.md`](../FINANCE/PAYMENTS/AML.md), [#14](https://github.com/Rivega42/indiahorizone/issues/14))
- [ ] **Скан паспорта** получен и сохранён в архиве клиента (зашифровано)
- [ ] Предоплата получена, чек выставлен
- [ ] Карточка клиента переведена в статус «оплачен»

### 6. Активная подготовка — sales + concierge

- [ ] Бронирования в IN запущены (Shivam / IN PVT LTD)
- [ ] Виза: клиент подал, уведомил статус
- [ ] Авиабилет: клиент купил, прислал PNR
- [ ] **Страховка получена** (полис) — критично для SOS / медицинских ЧП
- [ ] **Экстренные контакты заполнены** — ФИО + телефон родственника в РФ
- [ ] **Согласие на использование экстренных контактов** ([#55](https://github.com/Rivega42/indiahorizone/issues/55))
- [ ] **Согласие на фото/видео** ([#53](https://github.com/Rivega42/indiahorizone/issues/53)) — granular: для альбома / для маркетинга
- [ ] **Согласие на геолокацию** ([#54](https://github.com/Rivega42/indiahorizone/issues/54)) — для SOS обязательно, для шеринга опционально

### 7. За 7 дней до старта — concierge

- [ ] Все документы клиенту в Trip Dashboard загружены и доступны оффлайн
- [ ] Маршрут финализирован
- [ ] Гид(ы) назначены, контакты в карточке
- [ ] Бумажная карточка SOS подготовлена ([`docs/SOS/FALLBACK.md`](../SOS/FALLBACK.md))
- [ ] Финальный созвон с клиентом
- [ ] Чек-лист «к старту готов» проверен

### 8. За 1 день до старта — concierge

- [ ] Подтверждение трансфера от аэропорта (рейс на табло, время)
- [ ] Гид готов, в курсе ETA клиента
- [ ] Концьерж смены прилёта в курсе

### 9. Старт поездки — concierge

- [ ] Сообщение клиенту перед посадкой: «летите спокойно, мы встречаем»
- [ ] Push после посадки: «добро пожаловать, ваш гид у выхода»
- [ ] Подтверждение от гида: «забрал клиента»
- [ ] Перевод тикета в статус «в поездке»

## Чек-лист «к старту готов» (последний фильтр)

Печатается перед финальным созвоном:

```
КЛИЕНТ <ФИО>, поездка <даты>

Документы:
[ ] Паспорт + индийская виза (скан) загружены
[ ] Авиабилет (PDF + PNR) загружен
[ ] Полис страховки загружен
[ ] Бронирования отелей (ваучеры) — в Trip Dashboard
[ ] Программа дня — в Trip Dashboard

Согласия (152-ФЗ):
[ ] Базовое ПДн
[ ] Фото/видео (с указанием granular)
[ ] Геолокация
[ ] Экстренные контакты

Подготовка:
[ ] Гид назначен и проинструктирован
[ ] Бумажная SOS-карточка распечатана
[ ] AML-проверка пройдена
[ ] Финальный созвон проведён, клиент в курсе времени трансфера
[ ] Concierge на смене прилёта подтвердил готовность

ВСЕ ✅ — клиент готов к старту.
```

## Что делать если клиент пропадает

| Этап | Срок тишины | Действие |
|---|---|---|
| После лида | 3 дня | 1 контактное сообщение |
| После квиза | 5 дней | 1 контактное сообщение, потом архивация |
| После КП | 7 дней | Созвон с предложением скидки/уточнения |
| После договора, до предоплаты | 3 дня | Напоминание, предложение помощи с оплатой |
| После предоплаты, до старта | < 14 дней | Эскалация на Roman + Shivam |
| Не подал визу за 30 дней | — | Срочный созвон, риск форс-мажора |
| Не купил билет за 21 день | — | Срочный созвон |
