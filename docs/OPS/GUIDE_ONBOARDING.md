# Онбординг гида — наём, проверка, обучение, аттестация

> **Статус:** Draft v0.1. Часть EPIC 4 [#31](https://github.com/Rivega42/indiahorizone/issues/31).
> **Owner:** Shivam.
> Это HR-процесс расширения локальной сети. Связано с `STRUCTURE.md` (RACI), `GUIDE_CHECKLISTS.md` (что гид делает каждый день), `LEGAL/CONTRACTS/GUIDE_CONTRACT.md` (юридическая основа).

## Зачем это нужно

Главное узкое место фазы 3 — Shivam как единственная точка контроля качества гидов. Если онбординг не формализован:

- Невозможно расширять сеть без потери качества.
- При выходе Shivam в отпуск — никто не подхватит набор.
- Жалобы клиентов на «нашего гида» = жалобы на нас.

> **Рекомендация (опыт DMC в Индии):** в успешных DMC онбординг гида занимает 3–6 недель и включает обязательную поездку «гид-сопровождающий» рядом с senior-гидом. Мы в фазе 3 не можем себе позволить 6 недель, но 2 поездки «сопровождающим» — необходимый минимум, иначе теряем контроль над first-time-experience клиента. **Почему важно:** в Индии велика доля гидов, которые «знают город» технически, но не умеют считывать европейского клиента — и это не лечится инструктажем, только проживанием рядом.

```
Источник → Скрининг → Интервью Shivam → Тест-выезд → Стажировка → Аттестация → Договор → Активный гид
```

| Этап | Срок | Кто | Артефакт |
|---|---|---|---|
| 1. Источник | — | Shivam | Запись в HR-таблице |
| 2. Скрининг (документы, рекомендации) | ≤ 7 дней | Shivam | Чек-лист §1 ниже |
| 3. Интервью | 60–90 мин | Shivam | Заполненный профиль |
| 4. Тест-выезд (1 день) | — | Shivam + кандидат | Видео-фрагмент + отчёт |
| 5. Стажировка (2 поездки сопровождающим) | 7–14 дней | senior-гид | Письменный отзыв senior |

| 7. Договор | 1 день | Roman + finance | Подписанный контракт |
| 8. Активный гид | — | — | Профиль в CRM с тегом `active_guide` |

## §1. Скрининг — чек-лист

### Документы

- [ ] Удостоверение личности (Aadhaar / PAN) — проверка подлинности
- [ ] Сертификат гида (Government tourist guide license) — **обязательно** для регионов, где требуется по индийскому закону (ASI-памятники, нац.парки)
- [ ] Подтверждение опыта (фото-видео с предыдущими клиентами, рекомендации)
- [ ] Справка об отсутствии судимости (Police Clearance Certificate)
- [ ] Медсправка (общий допуск к работе)
- [ ] Контакты двух поручителей (другие гиды, отельеры, предыдущие работодатели)

> **Рекомендация:** не нанимаем гидов без Police Clearance даже если знаем лично. **Почему:** в Индии при инциденте с клиентом полиция первым делом проверяет background гида, и любая запись (даже не относящаяся к делу) превращается в недели задержки и репутационный удар. Лучше потерять кандидата на скрининге.

### Языки

| Язык | Уровень | Как проверяем |
|---|---|---|
| Русский | B2 минимум для русскоязычных гидов | 30-минутный диалог, ситуационные кейсы |
| Английский | B1 минимум для всех | Беседа |
| Хинди / локальные | по факту региона | Подтверждение Shivam |

### Регион

- Один гид = один регион primary + 1–2 secondary (где может работать).
- Для multi-region поездок — passing baton между гидами, а не один гид «куда угодно». **Почему:** локальная сеть и знание места теряются при гиде «на всю Индию». Качество в фазе 3 = глубина региона.

## §2. Интервью

90-минутный разговор. Структура:

1. **Биография и опыт** (15 мин) — история работы, типы клиентов
2. **Кейсы** (30 мин) — 3 ситуационные задачи:
   - Клиент почувствовал себя плохо в храме во время экскурсии — что делаешь?
   - Клиент возмущён ценой такси, угрожает написать «жалобу в полицию» — что делаешь?
   - Клиент-ребёнок отказывается продолжать программу из-за усталости — что делаешь?
3. **Ценности и стиль** (15 мин) — как видит роль гида
4. **Финансы и логистика** (15 мин) — как считает свою работу, ожидания
5. **Вопросы кандидата** (15 мин)

### Красные флаги

| Сигнал | Что значит |
|---|---|
| «У меня свои отели/магазины — буду водить туда» | Конфликт интересов, скрытая комиссия |
| «Зависит от чаевых» | Будет давить на клиента |
| «Если клиент пьян — это его проблема» | Нет понимания духа сопровождения |
| Не задаёт ни одного вопроса о клиентах | Не интересуется работой |
| Резко негативно о женщинах / о европейцах | Культурный риск |

> **Рекомендация:** красный флаг ≠ автоматический отказ, но требует двойной проверки на тест-выезде. **Почему:** в Индии прямота = нормальная коммуникация, и не каждый красный флаг — реальная проблема. Но «свои магазины» — это всегда финансовый конфликт, без исключений.

## §3. Тест-выезд (1 день)

Один день с Shivam (или senior-гидом) и тестовым клиентом или экскурсионной группой:

- Кандидат ведёт первую половину дня
- Shivam наблюдает молча, фиксирует
- Во второй половине — обратная связь, корректировки

**Оценочный лист (1–5):**

| Критерий | Что смотрим |
|---|---|
| Пунктуальность | Пришёл вовремя, готов |
| Внешний вид | Опрятность, форма |
| Внимание к клиенту | Замечает усталость, голод, дискомфорт |
| Коммуникация на русском | Естественность, юмор, тёплость |
| Знание места | Глубина, не википедия-факты |
| Безопасность | Анти-мошеннические практики, бдительность |
| Гибкость | Реакция на смену плана |
| Финансовая прозрачность | Честно ли называет цены, не предлагает «свои» места |

Минимум 4/5 в среднем + ни одной «1» → проходит дальше.

## §4. Стажировка (2 поездки сопровождающим)

- 2 поездки рядом с senior-гидом (не клиентом!)
- Кандидат не получает оплату как гид; получает суточные ([`docs/FINANCE/`](../FINANCE/), TBD ставки)
- Senior пишет письменный отзыв по 8 критериям §3 после каждой

**Что senior должен зафиксировать:**

- Как кандидат ведёт SOS-симуляцию (Shivam триггерит ложную тревогу — для теста)
- Как кандидат фотографирует/снимает кружки (для альбома клиента)
- Как кандидат ведёт debrief в конце дня

> **Рекомендация:** на стажировке обязательна одна симуляция SOS (например, «клиент сказал, что ему плохо»). **Почему:** в реальном инциденте 80% качества реакции зависит от первой минуты — оттренировать это можно только в безопасной среде. Без этого мы узнаём качество гида в момент реальной катастрофы.

## §5. Аттестация

После стажировки — 2–3 часа с Roman + Shivam:

- 30-минутный тест по знаниям (география, культура, логистика, безопасность)
- 60-минутный разбор кейсов из стажировки
- 30-минутный разбор playbooks SOS ([`docs/SOS/PLAYBOOKS/`](../SOS/PLAYBOOKS/))
- 30-минутное прохождение приложения гида (когда оно появится)

**Решение:**

- ✅ **Pass** — выходим на договор
- ⏸ **Conditional pass** — нужна доп. стажировка по конкретному пробелу (1 поездка)
- ❌ **Fail** — отказ с обратной связью

## §6. Договор

См. [`docs/LEGAL/CONTRACTS/GUIDE_CONTRACT.md`](../LEGAL/CONTRACTS/GUIDE_CONTRACT.md).

Кратко:
- Подрядный договор с IH IN PVT LTD
- Часть оплаты — фикс за день, часть — KPI (NPS клиента, ноль breach SLA)
- NDA + non-compete на работу с прямыми конкурентами в нашем сегменте
- TDS удерживаем как полагается

## §7. Активный гид — что у него есть

После подписания:

- Доступ к приложению гида (когда появится; в фазе 3 — Telegram канал «IH-guides» + чек-листы)
- Профиль в CRM с тегом `active_guide`, регионом, рейтингом
- График ротации (планирование на квартал)
- Контакты команды

## Постоянное развитие

### Ежемесячный 1:1 с Shivam (60 мин)

- Обзор клиентов, фидбэка
- Личные сложности, выгорание
- План развития (новый регион, новый язык, повышение ставки)

### Ежеквартальная аттестация

- Просмотр NPS, breach SLA, инцидентов
- Решение: повысить ставку / расширить регион / отправить на доп. обучение / расстаться

### Категории гидов (для CRM)

| Категория | Доступ | Ставка |
|---|---|---|
| `trainee` | Только сопровождение, без сольных поездок | суточные |
| `junior` | Соло-поездки, бюджет/средний сегмент | base |
| `senior` | Соло-поездки, премиум, тренинг trainee | base + 30% |
| `lead` | Управляет другими гидами в регионе | base + 50% + бонус за регион |

## Расторжение / временный вывод из ротации

### Триггеры немедленного вывода

- Жалоба клиента на безопасность / харассмент → расследование 48 часов
- Финансовый инцидент (клиент жалуется на навязчивые продажи / комиссию) → 48 часов
- 2 SOS-инцидента подряд с breach SLA → 7 дней
- Системный негативный фидбэк (NPS < 6 в трёх поездках подряд) → пересмотр

### Расставание

- Письменное уведомление за 30 дней (если не за критическое нарушение)
- Возврат материалов, удаление доступов в день расставания
- Заметка в CRM (для возможного повторного найма через год)

## Метрики

| Метрика | Цель |
|---|---|
| Скорость воронки (источник → активный) | ≤ 60 дней |
| Конверсия источник → активный | ≥ 40% |
| Средний NPS гида | ≥ 8 |
| % поездок с breach SLA по гиду | ≤ 3% |
| Удержание гида (год) | ≥ 70% |
