# SOS — резервный канал (SMS, звонок)

> Закрывает [#28](https://github.com/Rivega42/indiahorizone/issues/28). Часть EPIC 3 [#19](https://github.com/Rivega42/indiahorizone/issues/19).
> Статус: Draft v0.1.

## Зачем

В Индии интернет нестабильный. Клиент может оказаться без сети в момент ЧП (горы, пещеры, блэкаут, потеря телефона). SOS должен работать и в этом случае.

## Сценарии без интернета

### Сценарий 1: телефон есть, нет интернета (есть мобильная сеть)

1. Клиент видит в приложении бейдж «нет сети» — но кнопка SOS остаётся активной
2. Нажатие SOS → приложение **отправляет SMS** на наш короткий индийский номер
3. SMS-шлюз → API → создание тикета в `sos-svc`
4. Параллельно приложение **звонит** на наш номер concierge (через `tel:` → системный набор) — клиент сам говорит дежурному

### Сценарий 2: телефон утрачен / разряжен

1. У клиента в **бумажной карточке** при заезде (печатается онбордингом):
    - Индийский номер concierge: **+91 XXXX XXXXXX**
    - Резервный российский WhatsApp: **+7 XXXX XXXXXX**
    - Имя дежурного на смене сегодня
2. Клиент звонит с любого попавшегося телефона (отель, прохожий, полиция)

### Сценарий 3: интернет есть, но у дежурного упал

1. Дежурный не подтверждает SOS за SLA → `sos-svc` эскалирует резерву и founders
2. Дублирующий канал — Telegram-бот + автозвонок Twilio
3. См. [`SHIFTS.md`](./SHIFTS.md) — правило двух пар глаз

## SMS-протокол

Сообщение от приложения (системно):
```
SOS|<trip_id>|<user_id>|<lat>,<lng>|<idempotency_key>|<short_msg>
```

SMS-шлюз парсит и отправляет в `sos-svc` через POST с тем же `Idempotency-Key`.

## Бумажная карточка клиента

При финальной встрече (или вылете в Индию) клиент получает **печатную карточку** с:

- Индийский номер concierge 24/7
- Российский WhatsApp / номер
- Адрес и телефон ближайшего консульства РФ
- Адрес отеля (текущий)
- ФИО гида
- Свой PNR авиабилета

Карточка дублирует Trip Dashboard. Если потерял телефон — карточка спасает.

## Тестирование fallback

**Раз в месяц** — проверка резервного канала:

- [ ] SMS-шлюз принимает сообщение и создаёт тикет
- [ ] Звонок на индийский номер попадает дежурному
- [ ] Звонок на российский номер попадает на резерв
- [ ] Бумажная карточка раздаётся всем клиентам

Логирование тестов — в audit-log.
