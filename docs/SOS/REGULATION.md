## SLA первой реакции

Все времена — IST (India Standard Time, UTC+5:30), по локации клиента.

| Время суток (IST) | SLA первого ответа дежурного | Канал |
|---|---|---|
| 06:00 – 22:00 | ≤ **60 секунд** | Push + чат + автозвонок дежурному |
| 22:00 – 06:00 | ≤ **3 минут** | Push + чат + автозвонок дежурному + резервный звонок второму дежурному |

Breach SLA = **инцидент уровня P0**, фиксируется в audit-log, разбирается по [`POST_INCIDENT.md`](./POST_INCIDENT.md).

## Этапы реакции

```
1. Получение → 2. Подтверждение клиенту → 3. Диагностика → 4. Действие → 5. Закрытие
```

### 1. Получение (0 сек)

- `sos.triggered` event попадает в `sos-svc` (см. [`TECH.md`](./TECH.md))
- Push дежурному + сигнал в Telegram-бот команды + автоматический звонок (Twilio)
- Геолокация клиента и текущая активность маршрута прикладываются к тикету автоматически

### 2. Подтверждение клиенту (≤ 60 сек / ≤ 3 мин)

Дежурный нажимает «принял» в приложении → клиент получает push + текст в чате:
> «Мы рядом. Имя дежурного: <X>. Уже соединяемся.»

Это останавливает таймер SLA.

### 3. Диагностика (1–5 мин)

Дежурный задаёт три вопроса (скрипт первого ответа):
1. **Что случилось?** (медицина / полиция / транспорт / документы / другое)
2. **Где вы сейчас?** (геолокация уже есть, но подтверждаем)
3. **Вы в безопасности прямо сейчас?**

По типу ЧП — открывается соответствующий [`PLAYBOOKS/`](./PLAYBOOKS/).

### 4. Действие

По playbook'у. В каждом playbook'е — конкретные шаги, контакты, скрипты.

Параллельно:
- Если медицина / полиция / угроза жизни → второй дежурный + Shivam (founder) уведомлены
- Уведомление экстренных контактов клиента (если у клиента есть согласие, см. issue [#55](https://github.com/Rivega42/indiahorizone/issues/55))
- Гид клиента уведомлён, если он сейчас в смене

### 5. Закрытие

- Клиент подтверждает «всё в порядке» → тикет переходит в `resolved`
- В течение 48 часов — пост-инцидентный отчёт (см. [`POST_INCIDENT.md`](./POST_INCIDENT.md))

## Чек-лист дежурного при срабатывании (1 экран)

```
[ ] Принял (≤ SLA)
[ ] Подтвердил клиенту: "мы рядом, имя X"
[ ] Задал 3 вопроса диагностики
[ ] Открыл нужный playbook
[ ] Эскалировал второму дежурному (если медицина / полиция)
[ ] Уведомил гида (если в смене)
[ ] Уведомил экстренные контакты клиента (если согласие)
[ ] Зафиксировал ход в тикете каждые 5 минут
[ ] Дождался "всё в порядке" от клиента
[ ] Закрыл тикет, инициировал пост-инцидент в течение 48ч
```

## Кто всегда в курсе

| Тип ЧП | Кому уходит автоуведомление |
|---|---|
| Медицина (травма, отравление, госпитализация) | дежурный + второй дежурный + Shivam |
| Полиция / задержание | дежурный + второй дежурный + Roman + Shivam |
| Транспорт без угрозы | дежурный |
| Документы | дежурный |
| Любое неподтверждённое за SLA | вся команда + founders |
