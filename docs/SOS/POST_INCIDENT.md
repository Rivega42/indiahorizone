# SOS — пост-инцидентный разбор

> Закрывает [#30](https://github.com/Rivega42/indiahorizone/issues/30). Часть EPIC 3 [#19](https://github.com/Rivega42/indiahorizone/issues/19).
> Статус: Draft v0.1.

## Принцип

Каждый **реальный** SOS-инцидент (не FP) разбирается в течение **48 часов** после закрытия. Цель — учиться, обновлять playbook'и, не наказывать.

## Шаблон отчёта

Файл: `docs/SOS/INCIDENTS/YYYY-MM-DD-<short-slug>.md`

```markdown
# Инцидент: <короткое название>

- **Дата:** YYYY-MM-DD
- **Время начала:** HH:MM IST
- **Время закрытия:** HH:MM IST
- **Длительность:** Xч Yм
- **Клиент:** <анонимизировано — trip_id или alias>
- **Тип:** медицина / полиция / транспорт / документы / другое
- **Тяжесть:** P0 / P1 / P2

## Что произошло

(Хронологический рассказ, 5–15 строк.)

## Реакция команды

| Время | Кто | Что сделал |
|---|---|---|
| 22:14 | concierge-3 | принял SOS, ACK за 45 сек |
| 22:16 | гид Раджеш | выехал к клиенту |
| 22:30 | Shivam | подключился по медицинской ситуации |
| ... | | |

## Что хорошо

- (1–3 пункта)

## Что плохо / можно лучше

- (1–3 пункта)

## Action items

- [ ] Обновить `PLAYBOOKS/MEDICAL.md`: добавить контакт ближайшей больницы (owner: Shivam, до 2026-XX-YY)
- [ ] Поправить UX подтверждения (owner: Roman, issue #YY)
- [ ] ...

## Финансовое

- Расходы команды: ₹X (квитанции в архиве)
- Расходы клиента: ₹Y
- Со страховой клиента: получено / в ожидании / отказ

## Извлечённые уроки

(1–2 абзаца. Что мы поняли, что меняем в процессе.)

## Соответствие SLA

- SLA первого ответа: ≤ 60 сек / 3 мин — **выполнен / breach**
- Если breach — отдельный pункт в action items

## Анонимизация

ПДн клиента в этом отчёте: только `trip_id`. Имя, паспорт — нет.
```

## Сроки и owner

- **Авто-уведомление через 24 часа** после `sos.resolved` — concierge на смене получает напоминание написать отчёт
- **Owner отчёта** — дежурный, который вёл инцидент
- **Reviewer** — Shivam (для IN-инцидентов) + Roman (для всех)
- **Дедлайн закрытия отчёта** — 48 часов

## Куда сохраняется

- `docs/SOS/INCIDENTS/YYYY-MM-DD-*.md` (анонимизированно)
- audit-svc — полный лог тикета с ПДн (доступ ограничен)
- Action items → отдельные issues в репо с тегом `sos-followup`

## Обязательное обновление playbook'ов

Если в action items есть «обновить playbook X» — это **блокер** для закрытия отчёта. Reviewer не закрывает, пока playbook не обновлён или не создан issue с дедлайном.

## Метрики из всех инцидентов (квартал)

- Количество SOS / месяц
- Средний MTTR (mean time to resolve)
- % SLA breach
- Тренд по типам ЧП
- Топ извлечённых уроков

Эти метрики идут в дашборд founders ([`docs/ARCH/OBSERVABILITY/METRICS.md`](../ARCH/OBSERVABILITY/METRICS.md), [#74](https://github.com/Rivega42/indiahorizone/issues/74)).
