## Категории ПДн, которые мы обрабатываем

| Категория | Примеры | Источник |
|---|---|---|
| Базовые | ФИО, дата рождения, телефон, email | Квиз, договор |
| Документы для въезда | паспорт РФ (скан главной страницы), индийская виза | Загрузка клиентом |
| Контактные | адрес проживания РФ (для договора) | Договор |
| Финансовые | способ оплаты, реквизиты для возврата | Платёж |
| Особые предпочтения | диета, аллергии, медотметки | Квиз |
| Геолокация | координаты во время поездки (для SOS, шеринга) | Trip Dashboard приложение |
| Аудио/видео | «кружки», видеозвонки, чат-сообщения | Trip Dashboard |
| Экстренные контакты | ФИО и телефон родственника в РФ | Квиз |

**Не обрабатываем:** медданные сверх предпочтений (это отдельная категория с расширенным режимом — **сознательно избегаем**).

## Локализация (152-ФЗ ст. 18)

ПДн граждан РФ должны храниться на серверах в РФ.

**Решение:** все БД и хранилища файлов — в **российском облаке**:
- Postgres → Yandex Cloud Managed PostgreSQL
- S3 → Yandex Object Storage / Selectel S3
- Резервные копии — в РФ

Иностранные SaaS (OpenAI, Twilio, FCM/APNs) — только обезличенный трафик. ПДн в их системах **не сохраняем**.

См. [`docs/ARCH/DATA.md`](../ARCH/DATA.md) ([#62](https://github.com/Rivega42/indiahorizone/issues/62)).

## Реестр Роскомнадзора

Подача уведомления оператора ПДн (форма по 152-ФЗ):
- Подаётся **один раз**, до начала обработки
- Срок ответа Роскомнадзора: 30 дней
- Бесплатно

**Owner:** Roman + юрист.
**Дедлайн:** до начала продаж под ИХ-брендом в публичном маркетинге.

## Retention policy

Сколько храним каждую категорию:

| Категория | Срок | Основание |
|---|---|---|
| ФИО, контакты | 3 года после последней поездки | Возможные претензии |
| Скан паспорта | **Удаляется через 6 месяцев** после возвращения | Минимизация |
| Финансовые данные | 5 лет | Налоговое законодательство |
| Аудио/видео (кружки, чат) | 1 год после поездки, потом удаление или анонимизация | Дневник поездки = клиенту, наша копия удаляется |
| Геолокация треков | 30 дней | Только активная поездка |
| Audit-лог доступа к ПДн | 3 года | Compliance |

Удаление автоматическое — джоб в `clients-svc`, см. [`docs/OPS/POST_TRIP/ARCHIVE.md`](../OPS/POST_TRIP/ARCHIVE.md) ([#46](https://github.com/Rivega42/indiahorizone/issues/46)).

## Право клиента на удаление (152-ФЗ ст. 14)

Клиент может в любой момент потребовать:
- Доступ к своим ПДн (предоставляется в течение 10 рабочих дней)
- Удаление ПДн (срок — 7 рабочих дней)
- Прекращение обработки

Канал запроса:
- Email: `privacy@indiahorizone.ru` (TODO: настроить)
- Через личный кабинет — кнопка «удалить мои данные»

Регламент обработки запроса — [`docs/OPS/POST_TRIP/ARCHIVE.md`](../OPS/POST_TRIP/ARCHIVE.md).

## Регламент при запросе Роскомнадзора

1. Получили запрос → уведомление Roman + юриста в течение 24 ч
2. Сбор данных через audit-svc — кто, когда, что обрабатывал
3. Подготовка ответа в установленный срок (обычно 10–30 дней)
4. Юрист проверяет ответ перед отправкой

## Безопасность хранения

Минимально:
- TLS 1.3 на транспорте
- AES-256 шифрование at-rest на дисках облака
- Отдельное шифрование полей с паспортом / визой / финданными в БД
- Доступ к ПДн только через `clients-svc` API, никаких прямых SQL запросов из других сервисов
- 2FA для всех ролей с доступом к ПДн (concierge, sales, manager, finance, admin) — см. [`docs/ARCH/SECURITY/2FA.md`](../ARCH/SECURITY/2FA.md) ([#70](https://github.com/Rivega42/indiahorizone/issues/70))
- Audit-log всех просмотров и изменений — см. [`docs/ARCH/SECURITY/AUDIT_LOG.md`](../ARCH/SECURITY/AUDIT_LOG.md) ([#71](https://github.com/Rivega42/indiahorizone/issues/71))
