## Точки входа

1. **Push в 19:00 локально** → «Как прошёл день?» → тап → экран записи
2. **Кнопка «Записать кружок» на главном экране** Trip Dashboard
3. **Quick action** на иконке приложения (long-press)

### 2. Запись

```
┌──────────────────────────────────┐
│  ◀ Отмена                  60     │ <- таймер
│                                   │
│       ⊙ ── ── ── ── ── ── ⊙       │ <- круговая маска
│      │  📹 превью камеры │         │
│       ⊙ ── ── ── ── ── ── ⊙       │
│                                   │
│   [💬 переключить камеру]          │
│                                   │
│                                   │
│                                   │
│         (   ●   )                 │ <- большая кнопка hold-to-record
│                                   │
│       Удерживайте, чтобы           │
│       записать                     │
└──────────────────────────────────┘
```

При hold:
- Кнопка пульсирует
- Круг вокруг кнопки — таймер прогресса (заполняется по часовой стрелке)
- При отпускании раньше 5 сек — нет действия (минимум 5 сек для записи)
- При достижении 60 сек — auto-stop с лёгкой вибрацией

## Главный экран — статусы кружков

В Trip Dashboard на главной видна история:

```
[ Ваши кружки ]
☑ День 1 — отправлено
☑ День 2 — отправлено, concierge ответил
🔄 День 3 — в очереди (попытка 2/10)
☐ День 4 — записать
```

При тапе:
- ☑ — открывает запись (если consent.diary)
- 🔄 — детали с прогресс-баром, кнопка «отправить сейчас»
- ☐ — открывает запись

## Уведомления

| Когда | Текст | Поведение |
|---|---|---|
| 19:00 локально | «Как прошёл день <N>?» | Открывает экран записи |
| После доставки | «Concierge получил ваш кружок 💚» | На главном экране + чат |
| Если concierge ответил на кружок (не-стандартное) | «Maria ответила вам в чате» | Открывает чат |
| После 7 дней без записи кружка | (без push, чтобы не давить) | — |

## Privacy / тонкости

- Запись только при **активной поездке**, не до и не после
- Геометадата записи — отображается клиенту в просмотре («снято в Гоа, 19:08»)
- Клиент может **удалить свой кружок** в любой момент → удаляется и из дашборда concierge, и из S3 (с лёгкой задержкой)
- Если клиент отзывает согласие на использование — все его кружки **удаляются** в течение 7 дней

## Метрики UX

В аналитике (см. [`docs/ARCH/OBSERVABILITY/METRICS.md`](../ARCH/OBSERVABILITY/METRICS.md), [#74](https://github.com/Rivega42/indiahorizone/issues/74)):

- `circle.engagement.rate` — % дней, когда клиент записал кружок
- `circle.completion.rate` — % начатых записей, доведённых до отправки
- `circle.permission.granted.rate` — % клиентов, давших доступ к камере
- `circle.diary.consent.rate` — % давших согласие на использование в дневнике

Цель в фазе 3:
- ≥ 60% клиентов записывают ≥ 1 кружок
- ≥ 80% начатых записей отправлены
