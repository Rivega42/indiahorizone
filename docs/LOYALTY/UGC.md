# UGC и публичные отзывы

> **Статус:** Draft v0.1. Часть EPIC 8 [#86](https://github.com/Rivega42/indiahorizone/issues/86).
> **Owner:** Roman.
> **Метрика:** % клиентов с ≥ 1 видео-фидбэком ≥ 60%, % клиентов с публичным отзывом ≥ 40%.

## Зачем

Реклама в нашей нише не работает на холодную аудиторию. Работает только социальное доказательство — «мне сказал друг» или «я увидел отзыв реального человека».

Кружок (видео-фидбэк) и дневник поездки — это не только UX для клиента, но и сырьё для UGC. Программа лояльности здесь — про то, как **не передавить** на клиента и при этом получить материал, которым он сам захочет поделиться.

## Принципы

1. **Никогда не продавать «отзыв за скидку».** Это убивает доверие к отзыву и нарушает 152-ФЗ / закон о рекламе.
2. **Запрашивать согласие явно.** ПДн и согласие на использование изображения — отдельный документ, см. [`docs/LEGAL/CONSENTS/PDN.md`](../LEGAL/CONSENTS/PDN.md).
3. **Минимум трения.** Клиенту не нужно ничего верстать — мы публикуем, он одобряет.
4. **Качество > количество.** Лучше 10 настоящих кружков, чем 100 формальных «всё хорошо».

## Механики

### M1. Кружок-фидбэк (внутренний UGC)

См. [`docs/UX/VIDEO_CIRCLE.md`](../UX/VIDEO_CIRCLE.md).

Это не публичный материал по умолчанию. Это:
- Сырьё для дневника поездки клиента (его частная вещь).
- Сигнал concierge, что клиент удовлетворён или нет.
- **Потенциальный** материал для публикации — только с явного согласия.

### M2. Финальный кружок (опционально)

В последний день поездки — push: «Запишите 30-секундный итог поездки. Если хотите — мы спросим разрешение его опубликовать».

После записи — отдельный экран:

```
Ваш кружок сохранён в дневнике поездки.
Хотите ли вы, чтобы мы могли использовать его в наших материалах?
[ Нет, только для меня ]   [ Да, можно публиковать ]
```

«Да» = подписание согласия на использование изображения, см. [`docs/LEGAL/CONSENTS/PDN.md`](../LEGAL/CONSENTS/PDN.md).

### M3. Публичный отзыв на платформах

Через 7 дней после возвращения — push + email:

> «Если поездка вам понравилась, помогите следующим путешественникам — оставьте отзыв на одной из платформ. Это занимает 2 минуты, и для нас это важнее любой рекламы.»

Список платформ (приоритет — фаза 3):

1. Яндекс.Карты (если есть карточка организации)
2. Google Reviews
3. 2GIS
4. Профильные тревел-сообщества (Travelask, Otzovik) — опционально

Подсказка-подложка:
- Что упомянуть (1-2 предложения о маршруте, что запомнилось).
- Что не делать (упоминать конкретных гидов в негативе, юридически рискованных деталей).

**Не платим за отзыв. Не предлагаем взамен ничего.** Это критично — отзыв должен быть искренним, иначе работает против нас.

### M4. Дневник как готовый shareable артефакт

Дневник поездки = таймлайн с фото гида и (одобренными клиентом) кружками.

Возможности шаринга:

- **Приватная ссылка** (по умолчанию): доступна только тем, кому клиент отправил.
- **Скачать как видео-нарезку** (60–90 сек): автомонтаж, готов для соцсетей. Фаза 4.
- **Опубликовать на нашем сайте в галерее** (только с явного согласия): отдельный экран с предпросмотром, что будет опубликовано, и подписью под кейс.

### M5. Реактивация — «новости из Индии»

Раз в квартал — рассылка прошлым клиентам с фотографиями новых маршрутов / гайдов / историй гидов.

Цель: не «продать», а напомнить о существовании. Конверсия в повторную сделку и реферал — побочный эффект.

Условия:
- Клиент явно подписался на рассылку (opt-in, не opt-out).
- Контент — настоящий, не «промо».
- Отписка в один клик.

## Что мы НЕ делаем

- Не платим за отзывы.
- Не дарим скидку «за пятёрку на Яндексе».
- Не публикуем материал без явного согласия (даже «обезличенный»).
- Не пишем шаблонные отзывы и не просим клиента «подписать готовое».
- Не спамим по WhatsApp с просьбой оставить отзыв.

## Технология

### Фаза L0 (сейчас, ручной режим)

- Финальный созвон менеджера → ручной запрос на отзыв на платформах.
- Дневник — собирается вручную в Google Photos / Notion и отправляется ссылкой.

### Фаза L1 (в продукте)

- Дневник в Trip Dashboard, экран шаринга.
- Согласие на использование изображения — через Trip Dashboard.
- Учёт согласий в БД с возможностью отзыва клиентом в любой момент.
- Push-кампания «оставьте отзыв» через 7 дней — автоматическая.

### Поля в БД

| Поле | Описание |
|---|---|
| `consent_image_use` | bool, согласие на публикацию |
| `consent_image_scope` | enum: site_gallery / cases / social_media / none |
| `consent_image_granted_at` | timestamp |
| `consent_image_revoked_at` | timestamp (nullable) |
| `public_reviews` | массив ссылок на платформы, где оставлен отзыв |
