> **Owner:** Roman.

## Принцип фазы 3

**Не делаем кэшбек/баллы как у банков.** Это дорого в учёте, бессмысленно в нашем чеке (поездка ≠ кофе).

Делаем **три простых механики**, которые двигают конкретные метрики из [`docs/TZ/MVP_PHASE3.md`](../TZ/MVP_PHASE3.md):

| Механика | Метрика, которую двигает | Документ |
|---|---|---|
| Реферальная программа | % реферальных сделок ≥ 25% | [`REFERRAL.md`](./REFERRAL.md) |
| Repeat-привилегии | Возврат клиента (LTV) | [`REPEAT.md`](./REPEAT.md) |
| Дневник + публичный отзыв | UGC и публичные отзывы ≥ 40% | [`UGC.md`](./UGC.md) |

**Что осознанно НЕ делаем в фазе 3:**

- Накопительные баллы / уровни «золото-платина»
- Партнёрские карты (отели/рестораны)
- Скидки за подписку
- Геймификация (бейджи, leaderboard)

Это всё — на фазу 4, когда будет ≥ 200 платящих клиентов в год и есть смысл в системе.

## Юридический фундамент

Реферальные выплаты и скидки — публикуются как **часть оферты** или как отдельный документ-приложение к оферте.
См. [`docs/LEGAL/CONTRACTS/CLIENT_OFFER.md`](../LEGAL/CONTRACTS/CLIENT_OFFER.md) ([#50](https://github.com/Rivega42/indiahorizone/issues/50)).

Налоговая сторона: реферальный бонус для физлица — доход. Варианты:
- **Скидка на следующую поездку** — не считается доходом физлица, проще всего. **Рекомендация для фазы 3.**
- **Денежная выплата** — необходимо удержать НДФЛ или оформить как самозанятого. Только если клиент сам этого захочет.

См. [`docs/FINANCE/REFUNDS.md`](../FINANCE/REFUNDS.md) — общая логика возвратов и компенсаций.
