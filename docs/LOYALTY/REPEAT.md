## Зачем

JTBD J-C5 («поехать снова, но сильно проще»). Повторный клиент — самая выгодная сделка: CAC = 0, доверие построено, маршрут собирается быстрее.

В фазе 3 повторных клиентов мало (продукт молодой), но эта механика готовится **для двух целей**:

1. **Сократить трение** при второй покупке (анкета, договор, оплата).
2. **Дать символическое признание**, а не материальную скидку — потому что наш продукт не про скидки.

## Что НЕ делаем

- Уровни «золото-платина-бриллиант».
- Накопительные баллы.
- Подписочные модели «10% всем повторным».

Эти механики:
- Дороги в учёте.
- Размывают позиционирование (мы — concierge, не дискаунтер).
- Не двигают релевантную метрику для нашего чека.

## Что делаем

### 1. Авто-перенос профиля

После закрытия первой поездки клиент получает в Trip Dashboard:

- Сохранённую анкету (паспорт, диета, аллергии, темп, дети, опыт Индии).
- Историю предыдущих маршрутов с заметками гида и менеджера.
- Сохранённые контакты гидов, которые понравились (если оставлял ⭐).

При второй поездке менеджер видит профиль с пометкой «повторный» и работает по ускоренному скрипту (см. [`docs/OPS/CLIENT_ONBOARDING.md`](../OPS/CLIENT_ONBOARDING.md)).

### 2. Ускоренное оформление

Если у клиента валидные документы и реквизиты:

- КП за 1 рабочий день вместо 2.
- Договор и оплата — без повторной верификации.
- Можно зарезервировать слот гида до полной оплаты (риск берём на себя в обмен на скорость).

### 3. Символический жест на каждую N-ю поездку

| Поездка | Жест |
|---|---|
| 1-я | Стандартный сценарий + дневник в подарок |
| 2-я | Личный приветственный кружок от Shivam в день прилёта |
| 3-я | Бесплатный апгрейд одной ночи в маршруте + рукописная открытка |
| 5-я | Приглашение на закрытое мероприятие IndiaHorizone (founders' dinner раз в год) |

> **TODO founders:** утвердить. 3-я и 5-я — это «жест», не «бонус», и стоит дёшево, но создаёт эмоцию.

### 4. Приоритетный SOS

Повторный клиент попадает в более высокий tier SLA:

| Tier | SLA первого ответа (день / ночь) |
|---|---|
| Стандарт | ≤ 60 сек / ≤ 3 мин |
| Repeat (2+) | ≤ 45 сек / ≤ 2 мин |
| Repeat (5+) | прямой контакт Shivam при ЧП |

> **Замечание:** это **не** разный SLA по «качеству» (все клиенты получают полный стандарт), а **более жёсткий** для возвратных. См. [`docs/SOS/REGULATION.md`](../SOS/REGULATION.md).

### 5. Скидка — нет; апгрейд — да

Повторный клиент **не получает процентную скидку**, но получает приоритет на:
- Лимитированные slots (например, лучший гид на пиковый сезон).
- Апгрейд категории отеля при наличии доступности.
- Эксклюзивные маршруты (например, ранний доступ к новым программам).

Это сохраняет восприятие «премиум-concierge», а не «накопительная карта».

## Технология

### Фаза L0 (сейчас, ручной режим)

- Менеджер помечает в CRM статус «повторный» и применяет скрипт ускоренного онбординга.
- Жесты (открытка, апгрейд) — через Shivam, фиксируются вручную.

### Фаза L1 (в продукте)

- Профиль клиента с историей, сохранёнными предпочтениями.
- Автоматический подбор «той же команды» (если возможно) — гид, концieрж.
- Бейдж «повторный» в SOS-кейсах для concierge.

### Поля в CRM

| Поле | Описание |
|---|---|
| `trips_completed` | Счётчик закрытых поездок |
| `repeat_tier` | standard / repeat-2 / repeat-5 |
| `last_trip_at` | Дата последней поездки |
| `preferred_guide_ids` | Массив ID гидов с ⭐ |
| `gestures_history` | Лог жестов (что, когда, отмечено клиентом или нет) |

## Аналитика

Метрики для отслеживания:

- Доля повторных клиентов (target ≥ 25% к концу года).
- Время от заявки до КП у повторных vs новых.
- NPS повторных vs новых.
- Конверсия касания → оплата у повторных.
