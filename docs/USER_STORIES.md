# User stories — IndiaHorizone (фаза 3)

> **Статус:** Draft v0.1. Часть EPIC 1 [#1](https://github.com/Rivega42/indiahorizone/issues/1).
> Производное от [`docs/JTBD.md`](./JTBD.md). Каждая история ссылается на JTBD-родителя.
> **Owner:** Roman.

## Формат

Каждая история — `INVEST` (Independent, Negotiable, Valuable, Estimable, Small, Testable):

```
US-<role>-<n>. <Заголовок>
Как <роль>,
я хочу <действие>,
чтобы <цель>.

JTBD: <родитель из docs/JTBD.md>
Приоритет: P0 / P1 / P2
Где живёт: <модуль / документ>

Acceptance:
- [ ] критерий 1
- [ ] критерий 2
```

`P0` = без этого фаза 3 не закрывается.
`P1` = критично, но можно после P0.
`P2` = желательно, на горизонте фазы 4.

## Клиент

### US-CL-1. Получить КП по моему запросу за 2 рабочих дня

Как клиент,
я хочу после квиза и созвона получить готовое коммерческое предложение в одном PDF,
чтобы понять, что именно я покупаю и за сколько.

- **JTBD:** J-C1
- **Приоритет:** P0
- **Где живёт:** sales-flow, [`docs/BUSINESS_MODEL.md`](./BUSINESS_MODEL.md)

Acceptance:
- [ ] КП включает: маршрут по дням, отели, активности, цену с НДС, сроки бронирования
- [ ] Граница ответственности дублируется из оферты
- [ ] PDF + ссылка-просмотр в браузере (без логина)
- [ ] Срок: ≤ 2 рабочих дня от созвона

### US-CL-2. Подписать договор и оплатить онлайн

Как клиент,
я хочу принять оферту и оплатить (карта / СБП / расчётный счёт) без бумаги,
чтобы не ехать в офис и не возиться с принтером.

- **JTBD:** J-C1
- **Приоритет:** P0
- **Где живёт:** [`docs/LEGAL/CONTRACTS/CLIENT_OFFER.md`](./LEGAL/CONTRACTS/CLIENT_OFFER.md), [`docs/FINANCE/PAYMENTS/SCHEME.md`](./FINANCE/PAYMENTS/SCHEME.md)

Acceptance:
- [ ] Оферта подписывается через checkbox + подтверждение SMS/email
- [ ] Поддерживается карта, СБП, для юрлиц — счёт
- [ ] Чек/квитанция приходит в email и в Trip Dashboard
- [ ] Клиент видит статус «оплачено / частично / в работе»

### US-CL-3. Пройти квиз и анкету один раз

Как клиент,
я хочу один раз заполнить анкету о себе (паспорт, предпочтения, аллергии, бюджет),
чтобы менеджер не задавал те же вопросы по WhatsApp.

- **JTBD:** J-C1, J-C5
- **Приоритет:** P0
- **Где живёт:** [`docs/OPS/CLIENT_ONBOARDING.md`](./OPS/CLIENT_ONBOARDING.md)

Acceptance:
- [ ] Квиз ≤ 12 вопросов, время ≤ 5 минут
- [ ] Поля: паспортные данные, бюджет, диета, аллергии, темп, дети, опыт Индии
- [ ] Сохраняется в профиль клиента, переиспользуется во второй поездке

### US-CL-4. Открыть документы в Индии без интернета

Как клиент в Индии,
я хочу открыть паспорт / билет / ваучер / страховку без интернета,
чтобы пройти на reception, в полицию или к врачу.

- **JTBD:** J-C2, J-C3
- **Приоритет:** P0
- **Где живёт:** [`docs/UX/FEATURES/CORE.md`](./UX/FEATURES/CORE.md), `docs/ARCH/OFFLINE.md` (TBD)

Acceptance:
- [ ] Все документы предзагружены при первом подключении к Wi-Fi
- [ ] Доступ через 2 тапа с главного экрана
- [ ] При отсутствии сети — индикатор «офлайн, использую кеш»
- [ ] Карта региона работает офлайн

### US-CL-5. Видеть «сейчас» и «следующее» без поиска

Как клиент в поездке,
я хочу на главном экране видеть текущую активность и следующую,
чтобы не листать программу.

- **JTBD:** J-C2
- **Приоритет:** P0
- **Где живёт:** [`docs/UX/FEATURES/CORE.md`](./UX/FEATURES/CORE.md)

Acceptance:
- [ ] Главный экран = «Сейчас» + «Следующее» + таймер
- [ ] Push за 2 ч и за 30 мин до ключевого события
- [ ] Тап на событие → детали (адрес, контакт, заметка)

### US-CL-6. Записать видео-кружок одним тапом из push

Как клиент,
я хочу из вечернего push-уведомления одним тапом начать запись фидбэка кружком,
чтобы не открывать приложение и не искать кнопку.

- **JTBD:** J-C4
- **Приоритет:** P0 (ядро UGC-механики)
- **Где живёт:** [`docs/UX/VIDEO_CIRCLE.md`](./UX/VIDEO_CIRCLE.md), [`docs/ARCH/VIDEO_CIRCLE/`](./ARCH/VIDEO_CIRCLE/)

Acceptance:
- [ ] Push «Как прошёл день N?» в 19:00 локального времени
- [ ] Тап → экран записи (камера фронтальная, до 60 сек)
- [ ] Хранение офлайн при отсутствии сети, отправка позже
- [ ] Запись доступна в «дневнике поездки»

### US-CL-7. Поднять SOS hold-to-trigger

Как клиент в ЧП,
я хочу удержать SOS-кнопку 2 секунды и попасть на дежурного за 60 сек,
чтобы получить помощь без звонка через колл-центры.

- **JTBD:** J-C3
- **Приоритет:** P0
- **Где живёт:** [`docs/SOS/CONCEPT.md`](./SOS/CONCEPT.md), [`docs/SOS/TECH.md`](./SOS/TECH.md)

Acceptance:
- [ ] SOS-кнопка видна на каждом экране
- [ ] Hold 2 сек активирует, 10 сек окно отмены
- [ ] Передаётся геолокация и контекст поездки
- [ ] Резервный канал — SMS / звонок ([`docs/SOS/FALLBACK.md`](./SOS/FALLBACK.md))

### US-CL-8. Получить «дневник поездки» после возвращения

Как клиент,
я хочу через N дней после поездки получить готовый «дневник» — фото гида + мои кружки,
чтобы поделиться им и не разгребать WhatsApp.

- **JTBD:** J-C4
- **Приоритет:** P1
- **Где живёт:** UX (TBD)

Acceptance:
- [ ] Дневник = таймлайн дней с фото и кружками
- [ ] Доступен по приватной ссылке
- [ ] Можно экспортировать как видео-нарезку (фаза 4)

### US-CL-9. Перезаписаться второй раз без анкеты

Как повторный клиент,
я хочу собрать новую поездку, не проходя квалификацию заново,
чтобы потратить минимум времени на формальности.

- **JTBD:** J-C5
- **Приоритет:** P1
- **Где живёт:** [`docs/LOYALTY/`](./LOYALTY/), CRM

Acceptance:
- [ ] Профиль с предпочтениями переносится автоматически
- [ ] Договор/оплата в один клик (если данные не изменились)
- [ ] Менеджер видит «вторая поездка», прикладывает соответствующий tone of voice

### US-CL-10. Использовать реферальный код друга

Как клиент,
я хочу применить код друга и получить скидку, а друг — бонус,
чтобы привести знакомых без неловкости.

- **JTBD:** J-C5
- **Приоритет:** P1
- **Где живёт:** [`docs/LOYALTY/`](./LOYALTY/)

Acceptance:
- [ ] Реферальный код — у каждого клиента после первой поездки
- [ ] Код применяется на этапе КП
- [ ] Реферер видит статус («приглашён / купил / выехал»)

## Гид

### US-GD-1. Видеть смену и клиентов одной страницей

- [ ] Карточка клиента: имя, телефон, диета, аллергии, темп, заметки менеджера

### US-GD-2. Загрузить чек / расход в один тап

Как гид,
я хочу сфотографировать чек и привязать к клиенту/категории,

Acceptance:

- [ ] OCR — best-effort (фаза 4)
- [ ] Видно бухгалтеру в админке

### US-GD-3. Загрузить фото в альбом клиента

Как гид,
я хочу выбрать в галерее 10 фото и загрузить их в альбом конкретного клиента,

- **JTBD:** J-T2, J-C4

- **Где живёт:** UX гида (TBD)

Acceptance:
- [ ] Множественный выбор из системной галереи
- [ ] Привязка к клиенту/дню
- [ ] Загрузка в фоне с retry при плохом интернете

### US-GD-4. Принять SOS-эскалацию

Как гид,
я хочу получить пуш с SOS-сигналом моего клиента и геолокацией,

- **JTBD:** J-T6, J-C3
- **Приоритет:** P0

Acceptance:
- [ ] Пуш с типом инцидента и геолокацией
- [ ] Кнопка «принял / не могу»
- [ ] Эскалация при отсутствии ответа за SLA — на следующего по матрице

- Marketplace гидов как самостоятельный продукт — нет.
- Открытое API для турагентов — нет.
- Мобильное приложение для founders — на фазу 4.
