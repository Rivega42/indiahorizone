# Кружок: запись на устройстве

## UX

Триггер: вечерний push-уведомление в 19:00 по локальному времени клиента: «Как прошёл день?» — открывает экран записи кружка.

### Параметры записи

| Параметр | Значение | Обоснование |
|---|---|---|
| Длительность | до **60 секунд** (по умолчанию), с возможностью настройки 15/30/60 | Telegram-кружки = 60, привычно |
| Видео формат | MP4 H.264 baseline | Универсально, поддерживается всеми |
| Аудио | AAC 64 kbps mono | Достаточно для голоса |
| Разрешение | **480p** (854×480) | Баланс качества и размера для индийского интернета |
| FPS | 24 | Достаточно для лица |
| Размер на 60 сек | ~3–5 МБ после сжатия | См. [`COMPRESSION.md`](./COMPRESSION.md) |
| Камера | фронтальная по умолчанию, переключение на основную | Лицо клиента — основной кейс |
| Аспект | 9:16 (вертикальный) или 1:1 (квадрат) | Telegram-стиль |

### Поведение

1. Клиент открывает экран — преview камеры с круговой маской (как в Telegram)
2. Hold кнопки `[ ●  ]` для записи (не короткое нажатие — анти-FP)
3. Слайдер таймера (60 сек), цвет меняется при приближении к концу
4. Отпустил кнопку → запись остановлена → **превью с возможностью переснять**
5. На превью — `[ переснять ]` `[ отправить ]`
6. После отправки — кружок попадает в очередь (см. [`QUEUE.md`](./QUEUE.md))

### Доступ к камере и микрофону

- Запрос разрешений **explicit** при первом использовании, не на старте приложения
- Если отказ — UI «фидбэк только текстом», не упрашиваем
- В iOS: `NSCameraUsageDescription` + `NSMicrophoneUsageDescription` с понятным текстом
- В Android: runtime permissions

## Согласие на запись

Клиент видит **до записи** короткое уведомление:

> «Кружок попадёт в команду concierge для понимания, как прошёл день. С вашего согласия — также добавится в финальный «дневник поездки».»

С двумя чекбоксами (granular):
- [ ] Разрешаю использование в моём дневнике поездки
- [ ] Разрешаю использование анонимизированно в маркетинге IH

См. [`docs/LEGAL/CONSENTS/PHOTO_VIDEO.md`](../../LEGAL/CONSENTS/) (issue [#53](https://github.com/Rivega42/indiahorizone/issues/53)).

Согласие на запись = клик «отправить». Отказ от согласия = удаление кружка с устройства, не отправляем.

## Метаданные

К каждому кружку прикладывается:

```json
{
  "circle_id": "<uuid v4>",
  "trip_id": "<trip-uuid>",
  "client_id": "<user-uuid>",
  "day_number": 3,
  "recorded_at": "2026-04-25T19:08:00+05:30",
  "geo": { "lat": 15.4989, "lng": 73.8278 },
  "duration_sec": 47,
  "size_bytes": 2840000,
  "consent": { "diary": true, "marketing": false },
  "device": { "platform": "ios", "model": "iPhone 14", "app_version": "0.4.2" }
}
```

## Сохранение на устройстве (до отправки)

- Файл сохраняется в **app-private storage** (не в общую галерею)
- При успешной отправке + ack от сервера — файл **удаляется с устройства** через 24 часа (или сразу, если клиент включил «не хранить локально»)
- На устройстве — максимум 50 кружков в очереди / архиве
