## Почему это критично

Схема РФ ООО → IN PVT LTD через SWIFT работает. Но она хрупкая.

**Три риска:**

**1. Санкционный риск**
SWIFT-переводы из РФ в Индию работают через банки-посредники (Эмираты, Турция). Любое ужесточение санкционного режима — и цепочка рвётся. Срок перевода уже сейчас 3–7 дней, может вырасти до 3–4 недель.

**2. Кассовый разрыв**
Клиент заплатил в РФ → деньги дошли в Индию через 7–14 дней → отель нужно оплатить сейчас. Без оборотного резерва — кризис при каждом туре.

**3. Валютный риск**
Конвертация RUB→USD→INR на двух переходах. При скачке курса — маржа поездки исчезает.

## Минимальный план устойчивости

### 1. Оборотная подушка в IN

**Цель:** на счету IN PVT LTD всегда лежит сумма, покрывающая 2–3 тура вперёд.

Расчёт (грубо):
- Средняя стоимость наземки на 1 тур: ~150 000–250 000 ₽ (≈ $1 500–2 500)
- 3 тура: ~$5 000–7 000

Эта сумма не «заморожена» — она постоянно пополняется от входящих платежей. Это не резерв на случай кризиса, это **операционный буфер**.

### 2. Два канала перевода, не один

| Канал | Срок | Комиссия | Риск |
|---|---|---|---|
| SWIFT через банк РФ | 3–7 дней | $30–60 + ~0.5% | Основной |
| Wise / аналоги (через партнёра в нейтральной стране) | 1–3 дня | ~1–1.5% | Резервный |
| Crypto USDT → INR (через партнёра в Индии) | 1–2 часа | ~1–2% | Экстренный |

> **Важно:** crypto — не для ежедневных расчётов, а для экстренных случаев. Нужен проверенный партнёр в Индии с легальной конвертацией в INR.

### 3. Финансовый контроль

| Инструмент | Что делает |
|---|---|
| Еженедельная сверка | Сколько денег в РФ, сколько в IN, сколько в пути |
| Прогноз на 4 недели | Входящие платежи vs предстоящие расходы |
| Курсовой стоп-лосс | Если RUB/USD ухудшился на >X% → пересчёт чека для новых поездок |
| Бухгалтер РФ + бухгалтер IN | Синхронизация ежемесячно |

## Онлайн-платежи — следующий шаг

Сейчас: инвойс вручную → банковский перевод. Это работает, но:
- Клиент должен делать перевод сам → трение
- Нет автоматического учёта в системе
- Нет защиты от chargeback через систему

**Следующий шаг (M3 milstone, Slice I):**

| Вариант | Статус | Комментарий |
|---|---|---|
| ЮKassa / CloudPayments | Рекомендую | Работают с ИП/ООО в РФ, Россия |
| Robokassa | Уже в GrandHub | Быстрая интеграция по аналогии |
| Тинькофф Касса | Хорошая поддержка, API |
| Stripe | ❌ РФ не поддерживается напрямую | Через нерезидента — сложно |

**Что интегрируем:** ЮKassa или CloudPayments. Схема: клиент платит картой → деньги на счёт ООО → триггер в системе → статус поездки меняется автоматически.

## Межвалютная схема для платформы (фаза 4)

Когда появятся операторы в других странах:

```
Оператор из Марокко принимает оплату в EUR
        ↓
Платформа (IndiaHorizone OS) работает со счётом в нейтральной стране
        ↓
Оператор получает выплату в своей валюте
```

Это требует:
- Юридического лица в нейтральной юрисдикции (ОАЭ, Армения, Казахстан)
- Платёжного провайдера с мультивалютностью (Stripe для нерезидентов, Airwallex)
- Схемы налогообложения для SaaS-платежей

> **Сейчас это не нужно.** Фиксируем как требование к фазе 4.
