## Фаза 3 — Доказать модель (сейчас → Q3 2026)

### Цель
Все критичные процессы в коде. Команда не зависит от одного человека. Клиент получает experience, а не WhatsApp-переписку.

### Три обязательных блока

#### Блок А: SOS как система (не как герой)
**Почему сейчас:** обещание 24/7 уже дано клиентам. Оно держится на Шиваме. Это операционная бомба.

| # | Что | Результат |
|---|---|---|
| A1 | Нанять 2 concierge в Индии на дежурство (sleeping duty) | SOS не зависит от Шивама |
| A2 | Написать playbooks для 6 типов инцидентов | Дежурный знает следующий шаг, а не импровизирует |
| A3 | Собрать базу: госпитали, полиция, консульство, адвокаты, такси | Дежурный не гуглит в панике |
| A4 | SOS-кнопка в Trip Dashboard: геолокация + тикет + медкарта клиента | Одно нажатие — дежурный видит всё |
| A5 | Дашборд дежурного: скрипты, быстрые действия, таймер SLA | Дежурный не держит инструкции в голове |
| A6 | Эскалационная цепочка в системе (не в голове) | Шивам — последний уровень, не первый |

**Метрика успеха:** SOS SLA выполняется без звонка Шиваму в 95% случаев.

#### Блок Б: Кружки как экосистема
**Почему сейчас:** кружок — главный дифференциатор. Чем раньше накапливается контент, тем сильнее ров.

| # | Что | Результат |
|---|---|---|
| Б1 | Запись + загрузка кружка в нестабильной сети | 60% клиентов снимают хотя бы 1 кружок |
| Б2 | Push в 19:00 IST с персональным текстом | Ритуал, а не напоминание |
| Б3 | Кружок от гида (утренний брифинг) | Клиент видит лицо гида, снижается тревожность |
| Б4 | Дневник поездки: автосборка кружков + фото + маршрут | Sharable ссылка, которую клиент расшаривает сам |
| Б5 | Реферальный кружок со встроенной реф-ссылкой | Конверсия реферала ×3–5 vs текстового отзыва |

**Метрика успеха:** ≥60% клиентов снимают ≥3 кружка за поездку. ≥40% дают согласие на публикацию.

#### Блок В: Финансовая устойчивость
**Почему сейчас:** любой сбой SWIFT — и операционка встаёт.

| # | Что | Результат |
|---|---|---|
| В1 | Оборотный буфер $5–7k постоянно в IN | Нет кассового разрыва при задержке перевода |
| В2 | Резервный канал перевода (Wise/посредник) | Не зависим от одного банка |
| В3 | Онлайн-платежи ЮKassa вместо ручных инвойсов | Меньше трений для клиента, автоучёт в системе |
| В4 | Еженедельная финансовая сверка (процесс, не разовое) | Видим кассовый разрыв за 2–3 недели, а не в день Х |

**Метрика успеха:** нулевых задержек оплаты поставщикам из-за финансовых проблем.

## Фаза 4 — Concierge OS v1 (~Q4 2026 — Q1 2027)

### Цель
Первый внешний оператор запускается на нашей платформе.

### Что нужно сделать

#### Технически
| Что | Почему важно |
|---|---|
| Multi-tenant: tenant_id во всех моделях | Данные операторов изолированы |
| White-label тема Trip Dashboard | Оператор видит своё лого, не наше |
| SOS-as-a-Service: пул дежурных не привязан к оператору | Маленький оператор может купить наше дежурство |
| Operator onboarding wizard | Создай тенант, настрой квиз, добавь дежурных — за 1 день |
| Billing для операторов | Подписка + % транзакций |

#### Документально (упаковать IP)
| Документ | Что внутри |
|---|---|
| Operator Handbook | Методология ведения клиента: от квиза до дневника |
| SOS Playbooks v2 | Переносимые (не только Индия) |
| Guide Onboarding Kit | Шаблон найма и обучения местного гида |
| Client Journey Map | Полная карта касаний для адаптации под страну |

#### Операционно
| Что | Результат |
|---|---|
| 1 pilot partner (другая страна) | Proof of concept платформы |
| SOS-команда не зависит от Шивама | Готова обслуживать другого оператора |
| Публичная лента кружков запущена | Первый внешний контент-канал |

### KPI фазы 4
- Первый партнёр на платформе: Q1 2027
- MRR от SaaS: $2 000+
- NPS операторов: ≥70

## Фаза 5 — Рост платформы (~2027)

### Цель
3–5 операторов. Первый revenue из SaaS как основной источник.

| Направление | Что |
|---|---|
| Marketplace поставщиков | Отели, гиды, трансфер подают заявки сами. Операторы выбирают из проверенных. |
| Публичная лента кружков v2 | SEO-страницы по каждой локации. Органический трафик. |
| Certification program | Операторы проходят обучение → сертификат → появляются в каталоге |
| AI-ассистент дежурного SOS | Подсказывает следующий шаг по типу инцидента в реальном времени |
| Аналитика для операторов | LTV, CAC, NPS, SOS статистика — в одном дашборде |

## Фаза 6 — Отраслевой стандарт (~2028–2029)

### Цель
IndiaHorizone OS = то, что Shopify для e-commerce.

Нишевой туристический оператор в любой стране мира не строит Trip Dashboard, SOS, кружки, онбординг с нуля. Он берёт платформу и фокусируется на главном: людях, маршрутах и доверии.
