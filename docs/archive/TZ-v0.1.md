## 1. Обзор системы

### Цель

Единая платформа, которая объединяет клиента, гида, менеджера и управляющего вокруг одной поездки.

Для клиента — это самое удобное приложение путешествия: вся информация, документы, контакты, экстренная связь и ежедневный фидбэк в одном месте.

Для команды — операционный инструмент с CRM, аналитикой и управлением поездками.

### Ключевые принципы

| Принцип | Как реализован |
|---------|---------------|
| Оффлайн-first | PWA + локальный кеш, синхронизация при появлении сети |
| Mobile-first | Все основные сценарии — для смартфона |
| Русскоязычный UX | Весь интерфейс на русском |
| Минимум шагов | SOS, фидбэк, доп. услуга — максимум 2 тапа |
| Доверие через прозрачность | Клиент всегда знает, что происходит и что будет дальше |

## 2. Роли и права доступа

| Роль | Описание | Основной интерфейс |
|------|----------|-------------------|
| `client` | Путешественник | Trip Dashboard (мобильное приложение) |
| `guide` | Гид / локальный оператор | Панель гида (мобильное приложение) |
| `concierge` | Поддержка 24/7 | Веб + мобильный |
| `manager` | Менеджер по продажам | CRM (веб) |
| `finance` | Финансы / бухгалтер | Финансовый раздел (веб) |
| `admin` | Управляющий / основатель | Полный доступ (веб) |

Система RBAC: каждая роль видит только свои данные и функции.

## 3. Клиентский кабинет (Trip Dashboard)

Главный продукт для клиента. Доступен как мобильное приложение и PWA.

### 3.1 Документы и маршрут

- Все документы поездки: билеты, ваучеры, бронирования отелей, страховка, виза
- Полный маршрут с картой (оффлайн-доступ)
- Контакты: гид, concierge, посольство РФ, экстренные службы Индии
- Форматы: PDF, фото, ссылки

**Оффлайн:** весь раздел доступен без интернета, файлы кешируются при первом открытии.

### 3.2 Таймлайн дня

- «Сейчас» — текущая активность
- «Следующее» — ближайшее событие с таймером
- Полное расписание дня: трансферы, экскурсии, питание, свободное время
- Push-напоминания: выезд, чекин, лекарства, важные события
- Визуальный индикатор прогресса дня

### 3.3 Ежедневный фидбэк ⭐

Ключевая функция — обратная связь каждый день поездки.

**Сценарий:**
1. Вечером приходит push: «Как прошёл день X?»
2. Клиент отвечает: текст ИЛИ видео-кружок (до 60 сек)
3. Фидбэк сохраняется в историю и уходит команде
4. Concierge реагирует при необходимости

**Требования:**
- Запись видео прямо из приложения (нет выхода в камеру отдельно)
- Оффлайн-запись с отправкой при появлении сети
- Автоматическая сборка «дневника поездки» из всех фидбэков
- Оценка дня: 😔 😐 🙂 😊 🤩

### 3.4 SOS и экстренная связь

- Большая кнопка «SOS» всегда доступна (любой экран)
- Нажатие → выбор: звонок concierge / чат / автосообщение с геолокацией
- Резервный вариант: SMS на русский номер при отсутствии интернета
- Список экстренных контактов: посольство РФ, местная полиция, скорая, страховая

### 3.5 Запрос дополнительных услуг

- «Хочу добавить» → список категорий: трансфер, ужин, массаж, экскурсия, другое
- Краткое описание + предпочтительное время
- Подтверждение с ценой от менеджера/concierge
- История запросов

### 3.6 AI-ассистент по Индии

- Встроенный чат с LLM (GPT-4o)
- Контекст: маршрут клиента, его предпочтения, текущий день
- Умеет: рекомендовать, переводить, объяснять культурные нюансы, находить ответы по маршруту
- Не умеет: бронировать и менять поездку (это через concierge)

### 3.7 Переводчик и разговорник

- Встроенный переводчик: RU ↔ EN ↔ Hindi
- Голосовой ввод и воспроизведение перевода вслух
- Мини-разговорник: 50 ключевых фраз с произношением
- Оффлайн-режим для базовых фраз

### 3.8 Конвертер валют и учёт расходов

- Актуальный курс INR/RUB/USD (кешируется при наличии сети)
- Быстрый конвертер: ввожу сумму → вижу в трёх валютах
- Учёт личных расходов: категории (еда, транспорт, сувениры, другое)
- Итоги по дням и за всю поездку

### 3.9 Чек-листы

Готовые чек-листы с возможностью отмечать пункты:
- Упаковка перед поездкой
- Документы и виза
- Прививки и медицинские рекомендации
- Страховка
- Что взять в ручную кладь

### 3.10 Культурный гайд

- Дресс-код в храмах и священных местах
- Этикет: жесты, поведение за столом, фотографирование
- Чаевые: кому, сколько, в каких случаях
- «Что нельзя»: табу и ограничения
- Карта с отмеченными особыми зонами маршрута

### 3.11 Медгайд

- Рекомендованные прививки перед поездкой
- Ближайшие аптеки и больницы (на карте, оффлайн)
- Контакты посольства РФ и русскоязычных врачей
- Что делать при типичных проблемах: расстройство, тепловой удар, укус
- Контакты страховой компании клиента

### 3.12 Погода и риск-индикатор

- Погодный виджет для текущей локации
- Риск-индикатор дня: жара / муссон / пыль / ок
- Рекомендации: что взять с собой, как одеться

### 3.13 Шеринг с близкими

- Кнопка «Я в порядке» → уведомление выбранным контактам
- Шеринг геолокации в реальном времени (по желанию)
- Публичная ссылка на маршрут для близких (без персональных данных)

### 3.14 Фото-дневник поездки

- Автоматическая сборка из ежедневных видео-фидбэков и фото гида
- Клиент может добавлять свои фото
- Итоговый «дневник поездки» для скачивания/шеринга

### 3.15 Программа лояльности и рефералка

- Бонусные баллы за фидбэк, повторные поездки, рекомендации
- Реферальная ссылка: скидка другу + бонус клиенту
- История баллов и привилегий

### 3.16 Видеозвонок с concierge

- Кнопка «Позвонить concierge» → видеозвонок в приложении
- Доступна в любое время (concierge 24/7)
- Резервный вариант: аудиозвонок при слабом интернете

## 4. CRM — менеджеры и продажи

### 4.1 Карточка клиента

- Личные данные: имя, контакты, мессенджеры
- История поездок и коммуникации
- Предпочтения: стиль путешествия, бюджет, особенности
- Документы: копии паспорта, визы, страховки
- Теги и заметки

### 4.2 Воронка продаж

Этапы:
```
Лид → Квалифицирован → Квиз заполнен → КП отправлено → 
Переговоры → Предоплата → Активная подготовка → 
В поездке → Завершена → Повторная продажа
```

- Kanban-доска по этапам
- Список с фильтрами и сортировкой
- Автоматические триггеры при смене статуса

### 4.3 Квиз и онбординг клиента

- Клиент заполняет квиз: даты, бюджет, интересы, особые пожелания
- Результат квиза → автоматически в карточку лида
- Менеджер видит заполненный квиз и готовит КП

### 4.4 Коммуникация

- Встроенный чат с клиентом (Telegram-like)
- История переписки по всем каналам (Telegram, WhatsApp, Email)
- Шаблоны сообщений: приветствие, КП, напоминание, завершение
- Входящие запросы из всех каналов в одном месте

### 4.5 Управление поездкой

- Создание и редактирование маршрута
- Прикрепление документов к поездке
- Назначение гида
- Статусы оплаты
- Отправка материалов клиенту в Trip Dashboard

### 4.6 Задачи и напоминания

- Задачи по клиенту с дедлайном
- Автоматические напоминания: «через 3 дня вылет», «не ответили 2 дня»
- Назначение задач другим менеджерам

## 5. Панель гида

Мобильное приложение. Оффлайн-first.

### 5.1 Расписание и клиенты

- Расписание смен: текущий день и ближайшие
- Список клиентов на сегодня с карточкой каждого:
  - Имя, фото, особенности (аллергии, темп, языковой уровень)
  - Маршрут дня
  - Контакт concierge для связи

### 5.2 Коммуникация

- Чат с concierge
- Чат с клиентом (при необходимости)
- Push-уведомления: изменения расписания, сообщения

### 5.3 Отметки активности

- «Начал активность» / «Завершил активность» — кнопка + геометка
- Статус автоматически обновляется у менеджера и concierge
- Запись времени начала/конца для отчётности

### 5.4 Фото и видео

- Съёмка фото/видео прямо в приложении
- Автоматическая загрузка в альбом клиента (с его согласия)
- Оффлайн-накопление, синхронизация при сети

### 5.5 Учёт расходов

- Запись наличных/карточных расходов с категорией
- Фото чека
- Сверка с кассой в конце дня
- Итоговый отчёт по смене

### 5.6 SOS и форс-мажор

- Кнопка экстренной связи с concierge
- Автоматическая геометка при нажатии
- Резервный SMS

### 5.7 Рейтинг и выплаты

- История смен и оценок от клиентов
- Расчёт выплат за период
- Запрос выплаты через приложение

### 5.8 Оффлайн-доступ

- Весь маршрут дня и документы группы доступны без интернета
- Карты активностей (оффлайн)
- Кеш синхронизируется при появлении сети

## 6. Панель управляющих

### 6.1 Аналитика

**Воронка и продажи:**
- Количество лидов, конверсия по этапам воронки
- Выручка: план/факт, по периодам, по каналам
- CAC (стоимость привлечения клиента)
- LTV (пожизненная ценность клиента)
- Маржа по поездкам и маршрутам

**Качество сервиса:**
- NPS по поездкам
- Оценки гидов и маршрутов
- Топ проблем из фидбэков клиентов
- Время ответа concierge

**Когортный анализ:**
- Повторные покупки (когорты по первой поездке)
- Прогноз выручки на следующий квартал

### 6.2 Управление командой

- Список сотрудников с ролями и правами
- Метрики по менеджерам: лиды, конверсия, выручка, NPS
- Метрики по гидам: смены, оценки, расходы
- Аудит-лог: кто, что, когда изменил

### 6.3 Рассылки

- Сегментация клиентской базы: по статусу, дате поездки, интересам
- Рассылки через Telegram / Email / WhatsApp
- A/B-тесты писем
- Статистика открытий и кликов

### 6.4 Настройки продукта

- A/B-тесты квиза
- Управление шаблонами маршрутов
- Базовые цены и настройки расчёта стоимости
- Настройки автоматических уведомлений

### 6.5 Финансы

- Сводка по доходам и расходам
- Расходы по поездкам (гиды, отели, транспорт)
- Выплаты гидам
- Экспорт в CSV/Excel

## 7. AI-ассистент

### Клиентский AI (Trip Companion)

- Контекст: маршрут, предпочтения, текущий день поездки
- Отвечает на вопросы по Индии, маршруту, культуре
- Помогает с переводом и разговорником
- НЕ бронирует и не меняет поездку — направляет к concierge

### Внутренний AI (Manager Assistant)

- Помогает составить КП по заполненному квизу
- Анализирует фидбэки клиентов и выделяет паттерны
- Подсказывает ответы в переписке (черновики)
- Суммаризирует историю клиента перед звонком

## 9. Оффлайн-режим

Критично для клиентов в Индии: нестабильный интернет, роуминг, зоны без связи.

### Что работает без интернета

| Функция | Оффлайн |
|---------|---------|
| Документы и маршрут | ✅ Полностью |
| Карты маршрута | ✅ Скачиваются заранее |
| Контакты и экстренные номера | ✅ Всегда доступны |
| Разговорник (базовые фразы) | ✅ Встроен в приложение |
| Конвертер валют | ✅ Последний курс |
| Таймлайн дня | ✅ Из кеша |
| Запись фидбэка | ✅ Очередь, отправка при сети |
| Чат с concierge | ❌ Только при наличии сети |
| AI-ассистент | ❌ Только при наличии сети |

### Синхронизация

- При появлении сети: автоматическая синхронизация очереди
- Индикатор статуса сети и режима (оффлайн / онлайн / синхронизация)
- Конфликты разрешаются по принципу «сервер всегда прав»

## 10. Безопасность и данные

### Персональные данные

- Соответствие 152-ФЗ (российские клиенты) и GDPR (международные)
- Хранение документов (паспорта, визы): шифрование AES-256 at rest
- Передача данных: HTTPS / TLS 1.3
- Минимизация данных: собираем только необходимое

### Авторизация

- JWT-токены с ротацией
- Двухфакторная аутентификация для роли `admin` и `finance`
- Сессии с ограниченным временем жизни

### Аудит

- Аудит-лог: все изменения данных клиентов и финансовых операций
- Лог доступа к документам
- Хранение логов: 1 год

### Резервные копии

- Автоматические бэкапы БД: каждый час
- Хранение бэкапов: 30 дней
- Восстановление: тест 1 раз в квартал

```
┌─────────────────────────────────────────────────────────┐
│                    Клиентские интерфейсы                │

     │             │              │
┌────▼───┐   ┌────▼────┐   ┌────▼──────┐

│ Service│   │ Service │   │  Service  │
└────┬───┘   └────┬────┘   └────┬──────┘
     │             │              │
┌────▼─────────────▼──────────────▼──────┐
│           PostgreSQL (основная БД)     │
│     Redis (кеш, очереди, сессии)       │
│     S3 (файлы, фото, видео)            │
└────────────────────────────────────────┘
```

### Мобильное приложение

- React Native + Expo
- Локальное хранилище: AsyncStorage + SQLite (оффлайн)
- Синхронизация: background sync через Service Worker / Expo TaskManager
- Push-уведомления: FCM (Android) + APNs (iOS)

### Веб-портал

- Next.js 14 App Router
- PWA с Service Worker для оффлайн-режима
- Tailwind CSS + shadcn/ui
